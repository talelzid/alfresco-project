Déploiement du Serveur Alfresco sous Docker

Déployez une instance locale d'Alfresco Content Services (ACS) Community Edition 26.1 à l'aide du générateur officiel Docker d'Alfresco (stack complète : Repository, Share, PostgreSQL, Solr).

Prérequis

  • Docker installé et en cours d'exécution.
  • Node.js installé (nécessaire pour le générateur yo).
  • 16 Go de RAM alloués à Docker dans les paramètres de votre logiciel.
1 Installation de l'outil générateur

Le générateur yo automatise la création des fichiers de configuration Docker complexes.

npm install -g yo generator-alfresco-docker-installer
2 Création du projet

Créez un dossier de projet, placez-vous dedans, puis lancez le générateur :

mkdir mon-alfresco
cd mon-alfresco
yo alfresco-docker-installer

Répondez aux questions posées par le générateur comme suit :

4 Lancement du serveur

Dans le dossier du projet, lancez la construction et le démarrage des conteneurs :

docker compose up --build
Patience : La première exécution peut prendre 10 à 15 minutes. Attendez le message : Server startup in [NNNNN] milliseconds.
5 Accès à l'application
  • URLhttp://localhost:8084/share
  • Identifiantadmin
  • Mot de passeadmin

Aide au dépannage

  • Interface non trouvée : Assurez-vous d'ajouter /share à l'URL. L'URL racine pointe vers le moteur technique.
  • Serveur semble lent : Vérifiez que Docker a bien les 16 Go de RAM alloués dans ses paramètres.
  • Arrêter le serveur : Utilisez Ctrl + C dans le terminal, puis docker compose down pour nettoyer proprement.

Contingence et Communication

Plan de Contingence (Sauvegarde)

Chaque testeur dispose d'une machine virtuelle locale (VM) contenant une instance d'Alfresco (v. 7.4-7.4.1-1). Cette VM doit être utilisée comme environnement de secours en cas de panne ou d'indisponibilité prolongée du serveur de test principal.

Le fichier OVA nécessaire à la création de cette VM est disponible sur le canal du groupe Teams.
